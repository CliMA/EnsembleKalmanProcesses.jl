# EKP SLURM Pipeline

This `slurm-variant/` directory contains a SLURM job-dependency tree for running
an EnsembleKalmanProcesses.jl calibration on an HPC cluster.

Generated by the `slurm-pipeline-manager` skill.
**Run all scripts from the parent example directory** (the one containing your
Julia files), not from inside `slurm-variant/`.

---

## Quick start

### 1. Configure `slurm-variant/hpc_config.sh`

Open `slurm-variant/hpc_config.sh` and set at minimum:

| Variable | What to set |
|---|---|
| `JULIA_MODULE` | Julia module on your cluster, e.g. `julia/1.12.2` |
| `ACCOUNT` | Your SLURM account (`-A` flag) |
| `PARTITION` | Your partition, or leave empty for cluster default |
| `RUN_DATE` | **Pin this before every run** — it becomes the output directory name. Use `date +%Y-%m-%d` format. |

Resource dials (`*_TIME`, `*_MEM`, `*_CPUS`) have conservative defaults; tune
them for your forward model.

### 2. Precompile (once per environment)

```bash
bash slurm-variant/run_precompile.sh
```

Wait for it to finish before submitting the pipeline. Check with:

```bash
squeue -j <JID_printed_above>
```

You only need to rerun precompile when you update Julia packages (i.e., after
changing `Project.toml` or `Manifest.toml`).

### 3. Submit the pipeline

```bash
bash slurm-variant/run_pipeline.sh
```

This submits the full dependency tree:

```
setup → fwd[iter 0, array 1..N] → update[iter 0]
      → fwd[iter 1, array 1..N] → update[iter 1]
                ...
      → fwd[iter K-1, array] → update[K-1] → postprocess
```

Check the submitted jobs with `squeue -u $USER`.

### 4. Re-run postprocessing only

```bash
bash slurm-variant/run_postprocess.sh
```

Submits only `postprocess.sbatch` against the existing `output/$RUN_DATE/` tree.

---

## Dry run (test the pipeline without submitting)

```bash
bash slurm-variant/run_pipeline.sh --dry-run
```

Prints the exact `sbatch` commands with dependency flags, without submitting
anything. Use this to verify the tree looks correct before a real submission.

## Pipeline stages

| Script | Stage | Notes |
|---|---|---|
| `precompile.sbatch` | Julia `Pkg.instantiate` + `Pkg.precompile` | **Only job that precompiles.** Run once. |
| `setup.sbatch` | Build prior, initial ensemble, save `eki.jld2`, write iteration-0 member TOMLs | Runs `__SETUP_SCRIPT__` |
| `forward_map.sbatch` | Per-member forward model evaluation | Array job `1-N`; runs `__FORWARD_SCRIPT__ output_dir iteration member` |
| `update_ensemble.sbatch` | Collect outputs, call `update_ensemble!`, write next-iteration TOMLs | Runs `__UPDATE_SCRIPT__ output_dir iteration` |
| `postprocess.sbatch` | Plotting / analysis | Runs `__POSTPROCESS_SCRIPT__ output_dir` |

## Output directory layout

```
output/
  <RUN_DATE>/
    eki.jld2            # EKP object + prior + param_dict
    slurm/              # SLURM log files (.out / .err)
    iteration_000/
      member_001/
        parameters.toml   # constrained parameter values for this member
        output.jld2       # forward model output
      member_002/ ...
    iteration_001/ ...
```

## Key toggles

- **`JULIA_PKG_PRECOMPILE_AUTO=0`**: set in every run-stage sbatch to prevent
  concurrent array tasks from each trying to recompile Julia packages. Only
  `precompile.sbatch` compiles.
- **`--kill-on-invalid-dep=yes`**: kills downstream jobs if their dependency
  fails, rather than leaving them stuck in the queue.
- **`--array=1-$N_ENSEMBLE$FWD_ARRAY_THROTTLE`**: `FWD_ARRAY_THROTTLE` (e.g.
  `%50`) caps concurrent forward tasks; tune to match your cluster's fair-use
  limits.
- **`RUN_DATE`**: pin to a fixed string before each new run to keep output trees
  for different runs separate.

## Troubleshooting

**Jobs stuck in state PD (pending) for a long time:** check `squeue -j <JID>`
for the reason field; `Dependency` means the upstream job is still running;
`InvalidDependency` means it failed and was killed (check the `.err` log).

**A member's forward map failed:** look at
`output/$RUN_DATE/slurm/fwd_<array_jid>_<member>_iter<N>.err`. EKP will error on
the next update if fewer than 2 members succeed (by design).

**Package errors in run jobs but not precompile:** check that `JULIA_MODULE` in
`hpc_config.sh` matches the version used during precompile.

**Want to change N_ensemble or N_iterations?** Edit both `hpc_config.sh` and the
`#SBATCH --array=` line in `forward_map.sbatch` (the header is static; it cannot
read the config at parse time). Set a new `RUN_DATE` before resubmitting.
